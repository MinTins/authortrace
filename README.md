#### Цей репозиторій містить реалізацію практичної частини курсової роботи "КЛАСИФІКАЦІЯ ТЕКСТІВ ЗА ОЗНАКОЮ ШТУЧНОГО ПОХОДЖЕННЯ З ВИКОРИСТАННЯМ НЕЙРОННИХ МЕРЕЖ"

# AuthorTrace

**Гібридна нейромережева система детекції штучно згенерованих текстів.**

AuthorTrace визначає ймовірність того, що текст згенеровано великою мовною
моделлю (LLM). Система поєднує три класи сигналів — лінгвостатистичні,
перплексійні та семантичні — у фузійній нейромережі, а поверх неї працює
шар калібрування, що враховує особливості стилю сучасних LLM та зміщення
від машинного перекладу для україномовних входів.

---

## Можливості

* Двомовний аналіз: англійська (нативно) та українська (з автоматичним
  перекладом базового сигналу й аналізом стилістики на оригіналі).
* Виявлення стилю як класичних, так і сучасних великих мовних моделей.
* Інтерпретовані результати: внесок кожної групи ознак у вердикт,
  посегментний аналіз з виділенням підозрілих фрагментів, перелік
  спрацьованих правил калібрування.
* Чотири режими калібрування (`rules`, `lr`, `hybrid`, `none`) для
  різних сценаріїв використання.
* Інтерфейси: командний рядок, програмний API, веб-додаток на Streamlit.

## Архітектура

```
                ┌─────────────────────────────────────┐
   текст ──────▶│ Фузійна нейромережа                 │──┐
                │ (15 стилометр. + 8 перпл. + 768 сем.)│  │ raw_p
                └─────────────────────────────────────┘  │
                                                          ▼
                ┌─────────────────────────────────────┐  ┌──────────────┐
   текст ──────▶│ Розширені стилометричні ознаки      │─▶│ Калібратор   │
                │ (10 додаткових сигналів)            │  │ (rules / lr) │
                └─────────────────────────────────────┘  └──────────────┘
                                                                │
                                                                ▼
                                                          calibrated_p
```

Фузійна нейромережа приймає вектор з 791 ознак, що складається з:

* **15 базових стилометричних** — лексична різноманітність, структура
  речень, пунктуація, дискурсивні маркери, читабельність за Флешем;
* **8 перплексійних** — середня та локальна перплексія за `distilgpt2`,
  burstiness, ентропія розподілу токенів, рейтинг у top-k;
* **768 семантичних** — усереднений прихований стан тієї самої мовної
  моделі (одна модель обслуговує обидві групи, що робить систему
  легкою для запуску на CPU).

Шар калібрування пост-обробки додатково приймає на вхід:

* **10 розширених стилометричних ознак** — варіація довжин речень та
  абзаців, ентропія типів конекторів, hedge-density, паралельні
  конструкції, тріплет-перерахування «X, Y, Z», типографські тире,
  щільність віддієслівних іменників, рівномірність TTR, індекс
  формального регістру. Усі — двомовні (укр + англ), обчислюються
  над оригіналом тексту.

## Швидкий старт

### Встановлення

```bash
git clone <repository-url>
cd authortrace
pip install -r requirements.txt
```

### Командний рядок

```bash
# Стандартний режим (rule-based калібратор):
python scripts/demo_cli.py --file document.txt

# Детальний звіт зі стилометрією та спрацьованими правилами:
python scripts/demo_cli.py --file document.txt --verbose

# Україномовний текст (мова визначається автоматично):
python scripts/demo_cli.py --file ukrainian.txt

# Перемикання режиму калібратора:
python scripts/demo_cli.py --file doc.txt --mode rules    # за замовч.
python scripts/demo_cli.py --file doc.txt --mode lr       # log. regression
python scripts/demo_cli.py --file doc.txt --mode hybrid   # обидва
python scripts/demo_cli.py --file doc.txt --mode none     # без калібратора
```

### Програмний API

```python
from src.detector_v2 import AuthorTraceDetectorV2

detector = AuthorTraceDetectorV2(
    model_path="results/authortrace_model.pt",
    scaler_path="results/scaler.json",
    calibrator_path="results/calibrator.json",
    calibrator_mode="rules",
)

result = detector.analyze("Текст для аналізу...")
print(result["verdict"])                  # "AI" або "Human"
print(result["ai_probability"])           # фінальна ймовірність
print(result["raw_probability"])          # сирий вихід базової мережі
print(result["calibrated_probability"])   # після калібратора
print(result["extended_features"])        # 10 розширених стилометр. ознак
print(result["rules_fired"])              # спрацьовані правила
print(result["feature_contributions"])    # внески груп ознак
print(result["segments"])                 # посегментний аналіз
```

### Веб-інтерфейс

```bash
streamlit run app/streamlit_app.py
```

Інтерфейс надає галерею тестових прикладів (категорії «Людина»,
«Штучний текст», «Крайові випадки»), посегментну візуалізацію з
кольоровим градієнтом ймовірності та налаштування порогу
класифікації у бічній панелі.

## Режими калібрування

| Режим      | Опис                                                                       |
|------------|----------------------------------------------------------------------------|
| `rules`    | За замовчуванням. Інтерпретовані правила, що ловлять стиль сучасних LLM.   |
| `lr`       | Логістична регресія, навчена на калібрувальному корпусі.                   |
| `hybrid`   | Якщо обидва калібратори згодні щодо класу — середнє; інакше — rule-based.  |
| `none`     | Калібратор вимкнено; повертається сирий вихід фузійної мережі.             |

Rule-based режим базується на прозорих лінгвістичних правилах і не
залежить від конкретного калібрувального набору, тому добре
узагальнюється на нові домени. Логістичний режим може давати кращу
точність на текстах, розподіл яких відповідає калібрувальному корпусу.

## Повний конвеєр відтворення

Якщо потрібно відтворити навчання з нуля:

```bash
# 1. Підготовка тренувального корпусу (HC3).
python scripts/s1_prepare.py

# 2. Вилучення ознак (стилометрія + перплексія + семантика).
python scripts/s2_extract.py

# 3. Навчання фузійної нейромережі та оцінка на тестовій вибірці.
python scripts/s3_train.py

# 4. Тести стійкості до збурень (опечатки, перефраз, перемішування).
python scripts/s4_robustness.py

# 5. Навчання логістичного калібратора.
python scripts/s5_train_calibrator.py
```

Або одним викликом:

```bash
python scripts/run_pipeline.py
```

## Метрики

На тестовій вибірці HC3 (англомовний корпус, що використовувався для
навчання базової мережі) система демонструє точність понад 95% для
розпізнавання текстів класичних LLM.

На розширеному наборі, що охоплює тексти сучасних великих мовних моделей
та формальні україномовні людські тексти, калібрування пост-обробки
істотно покращує точність порівняно з сирим виходом фузійної мережі,
особливо в режимі `rules`.

---

## Обмеження

* **Стилометрично невідрізнимі тексти**: якщо людина пише в дуже
  формальному регістрі без характерних людських маркерів (живих
  спостережень, особистих оцінок, нерівних абзаців), система може
  помилково класифікувати текст як штучний.

* **Дуже короткі тексти (<60 слів)**: калібратор ослабляє свій вплив,
  оскільки стилометричні ознаки на малій вибірці шумні.

* **Швидко мінливі стилі**: словники конекторів, hedges та віддієслівних
  суфіксів налаштовані на українську й англійську мови та сучасний стан
  основних LLM. У разі суттєвої зміни стилю генерації або підтримки
  нових мов словники в `src/features/stylometric.py` потрібно
  розширити.
