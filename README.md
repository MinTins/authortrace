# AuthorTrace

**Гібридна нейромережева система детекції штучно згенерованих текстів**

AuthorTrace визначає ймовірність того, що текст згенеровано великою мовною
моделлю, поєднуючи три взаємодоповнювальні групи ознак у єдиній нейромережі
фузії. На відміну від типових детекторів, система пояснює власне рішення та
підтримує посегментну (часткову) детекцію.

---

## Ключові особливості

* **Гібридна архітектура з фузією ознак.** Один трансформер (`distilgpt2`)
  постачає одразу дві групи сигналів — перплексійні та семантичні; третя
  група — інтерпретовані стилометричні ознаки.
* **Проміжна фузія.** Кожна група ознак обробляється власним підкодувальником,
  після чого приховані представлення об'єднуються у спільному класифікаторі.
* **Пояснюваність.** Для кожного тексту обчислюється внесок кожної з трьох
  груп ознак у підсумковий вердикт (метод абляції гілок мережі).
* **Посегментна детекція.** Текст аналізується по фрагментах, що дозволяє
  локалізувати ділянки, найімовірніше згенеровані штучно.
* **Дослідження стійкості.** Передбачено оцінювання опірності детектора до
  спроб обходу шляхом «олюднення» згенерованого тексту.
* **Легкість.** Уся система працює на CPU без графічного прискорювача.

## Архітектура

```
            ┌─────────────────────────────────────────────┐
   текст ──▶│  Вилучення ознак                             │
            │   • стилометричні (15 ознак)                 │
            │   • перплексійні  (8 ознак, distilgpt2)      │
            │   • семантичні    (768, прихований стан LM)  │
            └───────────────┬─────────────────────────────┘
                            ▼
            ┌─────────────────────────────────────────────┐
            │  Гібридна нейромережа фузії (FusionMLP)      │
            │   стилометрія ─▶ підкодувальник ─┐           │
            │   перплексія  ─▶ підкодувальник ─┼─▶ фузія ─▶│─▶ P(штучний)
            │   семантика   ─▶ підкодувальник ─┘           │
            └─────────────────────────────────────────────┘
```

## Установлення

```bash
git clone <repository-url>
cd authortrace
pip install -r requirements.txt
```

Мінімальні вимоги: Python 3.10+, близько 2 ГБ вільного місця для моделей,
доступ до мережі для першого завантаження `distilgpt2` та корпусу HC3.

## Використання

### Наскрізний конвеєр

Конвеєр поділено на етапи, що дозволяє відновлювати роботу після переривання:

```bash
python scripts/s1_prepare.py      # 1. формування набору даних (HC3)
python scripts/s2_extract.py      # 2. вилучення ознак (виконати кілька разів)
python scripts/s3_train.py        # 3. навчання, оцінювання, графіки
python scripts/s4_robustness.py   # 4. дослідження стійкості до обходу
```

Скрипт `s2_extract.py` обробляє за один запуск частину текстів і зберігає
прогрес; його слід викликати повторно до повідомлення «ЗАВЕРШЕНО».

### Аналіз окремого тексту

```bash
python scripts/demo_cli.py --text "Текст для перевірки..."
python scripts/demo_cli.py --file document.txt
```

### Інтерактивний веб-інтерфейс

```bash
streamlit run app/streamlit_app.py
```

Веб-інтерфейс надає:

* **Галерею тестових прикладів** (11 курованих текстів у трьох категоріях —
  «Людина», «Штучний текст», крайові випадки). Кожен приклад
  супроводжується поясненням, чому він є показовим.
* **Аналіз довільного тексту** з імовірністю штучного походження.
* **Візуалізацію внеску** кожної з трьох груп ознак у вердикт.
* **Посегментний аналіз** з підсвічуванням підозрілих фрагментів.
* **Регульований поріг класифікації** у бічній панелі.

### Програмний інтерфейс

```python
from src.detector import AuthorTraceDetector

detector = AuthorTraceDetector(
    model_path="results/authortrace_model.pt",
    scaler_path="results/scaler.json",
)
result = detector.analyze("Текст для аналізу...")
print(result["verdict"], result["ai_probability"])
```

## Набір даних

Використовується корпус **HC3 (Human ChatGPT Comparison Corpus)** —
відкритий набір пар відповідей на однакові запитання, написаних людиною та
згенерованих моделлю ChatGPT. Корпус завантажується автоматично з
HuggingFace Hub.
